What APEX does

APEX simulates a structured executive debate over the viability of a biomedical research target. A research scout gathers evidence from PubMed, five domain specialists (science, technology, clinical, commercial, IP) assess the target from their respective vantage points, debate each other through a round of rebuttals, and a portfolio director synthesizes a GO / CONDITIONAL GO / NO-GO verdict with a confidence score.

The framework is domain-agnostic: swap the knowledge corpus and adjust the agent prompts and it can evaluate anything from a biotech target to a software architecture proposal. The default configuration demonstrates the protocol on publicly-studied knee osteoarthritis (KOA) drug targets.

After a GO verdict is accepted, a second graph runs in parallel to generate a Drug Development Plan (DDP) PDF — each specialist drafts their section and the director synthesizes the final report.

Agent coordination protocol

Phase What happens Why it matters
1. ScoutThree PubMed query variants, rank hits, PMC enrichmentGrounds debate in current peer-reviewed evidence
2. Parallel assessment5 advisors evaluate in parallel with role-specific RAGRemoves anchoring bias; reveals genuine disagreement
3. IP analysisDedicated IP Strategy Advisor runs Lens.org patent analysisIP posture often decides target selection
4. Adversarial debateRebuttals challenge at least one peer claim with evidenceSurfaces under-specified claims
5. SynthesisPortfolio Director writes weighted verdictSingle traceable decision — not vote count
6. Confidence loopRe-debate if confidence below thresholdReruns are evidence-informed

Quick start

git clone https://github.com/PatRoche2026/apex-framework.git
cd apex-framework
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Add your ANTHROPIC_API_KEY and VOYAGE_API_KEY
python -m rag.ingest
uvicorn server:app --reload --port 8000

Then try a full evaluation:

curl -X POST http://localhost:8000/evaluate \
  -H "Content-Type: application/json" \
  -d '{"gene": "MMP13", "indication": "Knee Osteoarthritis"}'
